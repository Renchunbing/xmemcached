Performance Comparison between Java Memcached Client

URL: http://blogjava.net/killme2008

There are several memcached client for java,include spymemcached, memcached-client-forjava(A optimiezed Java-MemCached), xmemcached etc. The performance numbers of the 3 open source memcached client are presented here to help you figure out the excellence of xmemcached in performance.

Where's the Graph?

If you are in a hurry, please scroll down to see the graphs first.

Test Scenario

A memcached server runs on a linux machine

A java client connect to memcached server,then start numbers of threads to repeat themself to store and retrive data from memcachd concurrently,write 30% and read 70%,ensure hit rate reach 70%

Test Environment

  • Software
    • Memcached Server
      • Version is 1.2.8,default install configuration.
      • Start with option "-p 12000 -m 4096"
    • Java Client
      • JVM Version: Sun JDK 1.6.0_07
      • xmemcached 1.1.1,default configuration
      • xmemcached 1.10,default configuration
      • xmemcached 2.3.1,default configuration
      • Java-MemCached 2.0.1,SockIOPool is default configuration,set compress threshold to 16K
      • JVM args: -server -Xmx1024m
      • Xmemcached and Spymemcached is nio based.So they have to be set operation timeout,they are both set to 3 seconds.
  • Hardware
    • Server
      • CPU: 8 x Intel(R) Xeon(R) 2.33GHz
      • O/S: Linux 2.6.9-67.ELsmp (redhat)
      • RAM: 16 GiB
      • NIC: Broadcom NetXtreme Gigabit Ethernet PCI express
    • Client (Hostname: Serpent)
      • CPU: 2 x Intel(R) Xeon(R) CPU 2.33GHz
      • O/S: Linux 2.6.9-55.ELxenU (redhat)
      • RAM: 16 GiB
      • NIC: Broadcom NetXtreme Gigabit Ethernet PCI express
